准备工作

  1. 准备一个U盘和软碟通刻录软件
  2. 一台服务器或者虚拟机
  3. 2个硬盘（或2个硬盘分区）。 OpenMediaVault需要单独的磁盘或分区用于操作系统安装和单独的磁盘或数据分区
  4. 最小 1GB RAM

安装前准备

本次机器为戴尔R420，制作了RAID10，将硬盘分为300g系统分区和8T数据分区

制作好RAID级别，并且保持机器有两个分区或者硬盘，只将机器安装到系统盘中，另一个数据盘中。

修改启动模式

OMV只支持UEFI方式启动，所以需要修改启动模式为UEFI。

将机器开机后，当看到右上角以下消息后按 键：

1
2

<F11> = UEFI Boot Manager（<F11> = UEFI 引导管理器）
还有一些F10,F12的信息

等待进入 Boot Manager

菜单介绍

选择Launch System Setup（启动系统设置），将启动模式修改为UEFI模式，然后按ESC退出到 Boot Manager界面

选择UEFI启动设备，Hard Drive C，等待从U盘启动

安装系统

安装文档：https://post.smzdm.com/p/av7z2564/

选择install

选择语言为第一个，尽量不要选择中文，会导致进系统后字符集错误，显示不全

选择位置用于设置时区

选择位置：选择china

选择键盘：美式英语

扫描网卡，如果有多网卡，会让你选择，请确认网线插得那个网口

可能会失败，点击继续，选择Configure network manually，去配置固定IP

如果你想使用固定IP，也可以在DHCP获取成功后，选择返回到此界面，再去选择Configure network manually

填写网络配置就可以了，

配置主机名，不改就直接回车

配置域名解析的名字，内网用不上，跳过

配置服务器密码，这个密码是登录服务器的不是web端后台的

确认密码

进行磁盘分区

选择安装的系统盘

等待自动分区和安装

配置软件包仓库，选择中国

配置软件源，选择清华源

没有代理，直接跳过

配置软件源中

安装完成,拔掉U盘，点继续进行重启

3.给文件夹划分权限

管理的逻辑为：创建一个文件夹，再创建用户和用户组，将用户加入到用户组中，在共享文件夹中的ACL控制中将用户组加入到文件夹中，这样就可以根据用户组队文件夹进行权限划分

创建用户组

创建用户

4.创建用户组对应的文件夹

创建文件夹介绍：

  • 名称：共享文件夹的显示名称，会显示在WINDOW共享界面
  • 设备：数据存放的分区位置
  • 路径：数据在分区中存放的路径，这个路径和OMV系统中的存放逻辑相同，在/src/设备名目录下
  • 权限：选择权限管理方案

选择权限方式：

  • 如果只想让这个组的人访问，就选择管理员读写，用户读写，其他不可访问
  • 如果想让所有人访问，就选择每个人读写

也可以创建子文件夹，可以指定在HRcenter下面，如：HRCenter/子目录

5.开启SMB共享目录
